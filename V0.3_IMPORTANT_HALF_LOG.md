# VoidClash v0.3 Important Half Log

Created: 2026-07-06

This is the more important half of the v0.3 mindmap to make first. The goal is to stabilize the core RTS loop before adding more flavor. If these are solid, the rest of v0.3 becomes much easier to tune.

## 1. Critical Fixes

### Worker mining stuck bug

- Workers sometimes get stuck at mineral nodes.
- Workers can stop harvesting without being ordered.
- Add stall detection for worker mining, movement to minerals, and movement to dropoff.
- If a mineral or dropoff becomes unreachable, retarget to another valid one.
- Add debug logging or temporary state labels for worker harvest states during testing.

### Construction reliability

- Check whether a worker can path to a build site before assigning it.
- Make selected workers able to assist unfinished buildings more reliably.
- Recover idle builders that still have an unfinished assigned site.
- Make worker return-to-mining after construction more dependable.

### Pathfinding polish

- Improve unit handling around building corners.
- Reduce worker clumping at mineral lines and dropoff buildings.
- Improve movement behavior when units are routed through tight base layouts.

## 2. Economy

### Worker behavior

- Add better idle worker detection.
- Add an idle worker indicator or button.
- Make workers automatically resume useful work after construction or path failure.

### Expansions

- Make second Command Center placement easier to understand.
- Make expansion mineral locations more readable.
- Later v0.3 pass: let the enemy expand in harder missions.

### Balance

- Tune mineral income after worker reliability is fixed.
- Tune build times after economy flow is stable.
- Tune supply pressure so it feels RTS-like but not annoying.

## 3. Building System

### Terran-style building

- Improve placement grid readability.
- Show blocked placement tiles more clearly.
- Add placement range from worker if it improves clarity.
- Consider building rotation only after placement reliability is stable.

### Building commands

- Add cancel construction with partial refund.
- Improve rally point UI feedback.
- Improve land/lift feedback.
- Consider a repair command after worker construction bugs are fixed.

## 4. Missions

### Campaign polish

- Playtest and tune all six v0.2 missions.
- Add clearer objectives in mission briefings and HUD messages.
- Add mission victory text.
- Improve mission select progress display.

### Mission types to prioritize

- Holdout defense.
- Destroy boss.
- Expand-and-survive.
- Base race mission.

## 5. UI and UX

### HUD polish

- Add idle worker indicator.
- Make build progress easier to read.
- Improve supply warning.
- Improve "not enough minerals" and blocked placement feedback.

### Command card

- Make hotkeys easier to read.
- Disable impossible actions with a clearer reason.
- Keep buttons compact and useful for repeated RTS play.

## 6. AI and Testing

### AI stability

- Improve AI worker recovery.
- Improve AI supply depot timing.
- Make wave compositions more varied after economy is stable.
- Improve boss escort behavior for boss missions.

### Tests and release checks

- Add a reproducible test for the worker mining stuck bug.
- Smoke test all six missions.
- Test worker build/harvest loops.
- Test lift-off and landing after placement changes.
- Always extract-test the release zip before upload.
- Include SHA256 in release notes.

## Deferred Half

These are still good v0.3 ideas, but they come after the core loop is stable:

- Bigger visual polish.
- New sound sets and voice acknowledgements.
- Extra unit silhouette work.
- New optional buildings.
- More story flavor.
- More advanced difficulty personalities.
